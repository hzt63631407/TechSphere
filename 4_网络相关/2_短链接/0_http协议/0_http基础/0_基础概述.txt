



一、http特点：
1.无链接，一次处理一个请求
2.无状态 对于事物没有记忆，没有记录前面的信息。
短连接：----服务器不能主动给客户端发消息





二.什么是Http协议无状态协议?怎么解决Http协议无状态协议?

(1)、(短链接，连接之后断开)
同一个客户端的这次请求和上次请求是没有对应关系
对http服务器来说，它并不知道这两个请求来自同一个客户端。


(2)、无状态协议解决办法：
1、Cookie
2、通过Session会话保存。

3.keepalive是短时间内多个http请求可复用同一个tcp连接。
从HTTP/1.1起，默认都开启了Keep-Alive，保持连接特性，
简单地说，当一个网页打开完成后，
客户端和服务器之间用于传输HTTP数据的TCP连接不会关闭，
如果客户端再次访问这个服务器上的网页，会继续使用这一条已经建立的连接。
Keep-Alive不会永久保持连接，它有一个保持时间，
可以在不同的服务器软件（如Apache）中设定这个时间。
(tcp不断开，http断开)



4.Http/1.1和一部分的Http/1.0持久连接（http keep-alive）
持久连接的特点是，只要任意一端没有明确提出断开连接，则保持TCP连接状态。

HTTP1.0会直接断开，HTTP1.1 默认情况下建立 TCP 连接不会断开，
只有在请求报头中声明 Connection: close 才会在请求完成后关闭连接。




5.Keep-Alive对每个请求仍然要单独发 header，就是除了真正的数据部分外，
服务器和客户端还要大量交换 HTTP header，信息交换效率很低。

WebSocket 解决的第一个问题是，通过第一个 HTTP request 建立了 TCP 连接之后，
之后的交换数据都不需要再发 HTTP request了
