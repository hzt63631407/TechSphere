



一、SSL（Secure Sockets Layer）主要运行在应用层（Application Layer）和
传输层（Transport Layer）之间，它是为网络通信提供安全和数据完整性的一种安全协议。

SSL，全称为Secure Sockets Layer，是一种安全协议，目的是为网络通信提供安全及数据完整性保障。
SSL 使用加密技术，确保数据在两个系统（例如服务器和客户端）间传送过程中不被窃取或篡改。
它在数据包传送从一个网络节点到另一个网络节点的过程中提供了保密性和数据完整性。

SSL协议主要有以下功能：
加密保护数据：SSL通过对交换的数据进行加密，防止数据在传输过程中被第三方获取。
身份验证：SSL使用数字证书进行身份验证，确保数据的发送方和接收方的真实性。
保护数据完整性：在数据传输过程中，SSL可以防止数据被篡改，确保数据的完整性。

ssl即有通信加密，也有内容加密。






二、http+ssl=https 可以这么理解吗？

是的，你的理解是正确的。

HTTPS 实际上是在 HTTP 上构建了 SSL/TLS 协议，
为数据传输过程提供了加密、数据完整性和服务器以及客户端身份验证等安全保障。
简单来说，它就是通过 SSL/TLS 协议对 HTTP 的内容进行了加密处理，使得数据在传输过程中更加安全。

HTTP 是明文传输，而 HTTPS 则通过加密手段保护了传输数据的隐私与完整性。
此加密过程的原理是：首先客户端和服务器先通过 SSL/TLS 的握手过程建立加密参数，
之后的通讯数据就使用这些参数进行加密，从而完成了 HTTP 数据的加密获得 HTTPS。
因此，“HTTP+SSL=HTTPS”这种表述是可以被接受的。然而要注意，虽然这种表述很直观，
但 SSL/TLS 的作用远不止于此，并且 HTTPS 协议还包含了对服务器身份的验证等环节。
